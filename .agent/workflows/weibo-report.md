# 微博热搜分析报告生成流程

## 前置条件
- 确保网络连接正常
- 百度搜索API和微博热搜API可用

---

## 执行步骤

### 步骤1: 获取微博热搜数据
// turbo
1. 切换到项目目录并运行数据获取脚本：
   ```bash
   cd "e:\My Docment\Celeste\Coding\微博热搜分析" && node fetch_data.js
   ```
   - 该脚本会调用微博热搜API获取10个热搜话题
   - 数据保存到 `analysis_data.json`

### 步骤2: 读取并验证数据
2. 读取 `analysis_data.json` 文件，确认包含以下内容：
   - 热搜话题列表（至少10个）
   - 每个话题的热度值和标签

### 步骤3: 使用百度搜索API调研背景（必做）
3. **对每个热搜话题**，必须调用百度搜索API获取背景信息：

   **API调用方式**（使用 run_command 执行curl）：
   ```bash
   curl --location "https://qianfan.baidubce.com/v2/ai_search/web_search" --header "X-Appbuilder-Authorization: Bearer bce-v3/ALTAK-L6TPvDXqOGEqEIB2Ogh0z/4432bd66294ce9b19fdca57204bd2024c8e40db6" --header "Content-Type: application/json" --data "{\"messages\":[{\"content\":\"[热搜话题关键词]\",\"role\":\"user\"}]}"
   ```

   **⚠️ 严格要求**：
   - ❌ 禁止基于推测或常识编写背景
   - ❌ 禁止虚构事件细节或时间线
   - ✅ 必须引用真实的搜索来源
   - ✅ 背景描述必须与搜索结果一致
   - ✅ 如果搜索结果有限，要如实说明
   - ✅ 所有背景信息必须有可追溯的来源URL

### 步骤4: 对每个话题进行深度分析
4. 针对每个热搜话题，生成以下分析内容：

   **A. 背景来龙去脉**（基于步骤3的搜索结果）：
   - 搜索来源：列出使用的关键词和返回结果数量
   - 事件详情：引用搜索结果原文，标注来源URL
   - 事件时间线：根据搜索结果梳理
   - 社会影响：引用搜索结果中的描述

   **B. 深度分析**：
   - 用户痛点：暴露的问题和需求
   - 市场机会：潜在的商业价值
   - 创新方向：技术/产品创新点

   **C. 软件产品创意**（每个话题必须2个）：
   - ✅ 网页应用/小程序
   - ✅ 移动APP
   - ✅ 在线平台/工具
   - ✅ 数据可视化系统
   - ❌ 禁止硬件产品
   - ❌ 禁止实体商品
   - ❌ 禁止纯线下服务

   **产品方案必须包含**：
   - 产品名称和定位
   - 核心功能描述
   - 目标用户群体
   - 市场潜力评估
   - 技术实现方案

### 步骤5: 生成HTML报告
5. 使用 `write_to_file` 工具生成最终报告：
   - **文件命名**: `weibo_analysis_report_YYYYMMDD.html`（例如：weibo_analysis_report_20260121.html）
   - **保存路径**: `e:\My Docment\Celeste\Coding\微博热搜分析\`

   **报告结构**：
   1. 数据概览 - 热搜数据统计
   2. 话题深度分析 - 每个热搜的完整分析
      - 背景来龙去脉（含来源链接）
      - 深度分析
      - 软件产品创意（2个/话题）
   3. 总结与建议 - 优先级排序和行动计划

   **HTML设计要求**：
   - 现代化响应式设计
   - 清晰的信息层级
   - 美观的视觉呈现（渐变、卡片、阴影等）
   - 移动端友好
